# Data Storage Specification

## Overview

All captured data is stored locally in a date-based directory hierarchy. No data is transmitted externally during capture.

## Directory Structure

```
{output_dir}/
├── {YYYY-MM-DD}/
│   ├── {YYYY-MM-DD}.log            # Daily log file (keyboard + event entries)
│   ├── click_*.webp                 # Click screenshots
│   ├── dblclick_*.webp              # Double-click screenshots
│   ├── drag_*.webp                  # Drag screenshots
│   ├── mic_*_{app}_dur*.wav          # Microphone audio segments (per-session)
│   ├── click_*.txt                  # LLM analysis for click screenshots
│   ├── dblclick_*.txt               # LLM analysis for double-click screenshots
│   └── drag_*.txt                   # LLM analysis for drag screenshots
│
├── {YYYY-MM-DD}/
│   └── ...
│
└── reports/
    ├── {YYYY-MM-DD}.md              # Daily activity report
    └── {YYYY-MM-DD}_images.md       # Screenshot analysis detail report
```

## Default Output Directory

`~/auto-capture`

Configurable via:
1. `--dir` / `-d` CLI argument (highest priority)
2. `OPENCAPTURE_OUTPUT_DIR` environment variable
3. `capture.output_dir` in config YAML
4. Default: `~/auto-capture`

## Date Directories

- One directory per calendar day, named `YYYY-MM-DD`
- Created automatically on the first captured event of the day
- If the system clock crosses midnight during a capture session, new events go into the new date directory

## Log File

- One log file per day: `{YYYY-MM-DD}.log`
- Append-only; never truncated or overwritten during capture
- Shared by keyboard logging, screenshot entries, and audio entries
- Encoding: UTF-8
- See [keyboard-logging](keyboard-logging.md), [screenshot-capture](screenshot-capture.md), and [microphone-capture](microphone-capture.md) specs for entry formats

## Analysis Files

- Generated by the LLM analyzer in analysis mode (not during capture)
- One `.txt` file per screenshot, with the same base name as the image file
- Content is the LLM's analysis of the screenshot

## Report Files

- Generated by the report generator in analysis mode
- Stored in `{output_dir}/reports/`
- `{YYYY-MM-DD}.md` — daily summary report
- `{YYYY-MM-DD}_images.md` — detailed screenshot analysis report
